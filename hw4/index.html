Part 3: Handling collisions with other objects

In this section, we implement collision object between the point masses and spheres and planes. This allows us to model interesting behavior such as the cloth drooping over the sphere.
First, for the sphere, we check if the point mass's position is inside the sphere. If it is, we appropriately adjust its position based on its previous position and the current position, as well as a small correction (which is also slightly adjusted by friction).
Then, for a plane, we check if the pointmass passes through the plane between its last position and current position, and if it does, we adjust its position in a similar way to the sphere, using the last and current position, as well as a small friction factor, to keep its current position on the same side of the plane as its last position.
Lastly, in the simulation we check every pointmass to see if it collided with an object, which would update its position to correct the intersection.

Part 4: Handling self-collisions

In this section, we implement an efficient self-collision algorithm to prevent the cloth from interesecting with itself. We do this by moving two points apart if they are too close to one another.
To keep this efficient, we hash the space into multiple cubes, and mapping points which lie within the same cube to the same array. This way, we don't have to check every point against every other point. Then, for each hash, we loop over the points in it and check if they are too close, applying moving them apart if they are. We implemented the hashing function by floor dividing the pointmass's position by the dimensions of the cube, to find which cube it would lie in, and converting the 3d coordinate corresponding to the cube with 3 distinct primes (3 digit ones though larger would probably work better). This way, each hash uniquely corresponds to each cube.

Explain in your own words what is a shader program and how vertex and fragment shaders work together to create lighting and material effects.

The shader program is a program that calculates a 4D vector output (which can include stuff such as color) given the appropriate inputs (such as camera position, surface normals, etc.), which runs in parallel, on the gpu, to the rest of the code to significantly speed up the rendering process. Shaders usually consist of both a vertex and fragment shader, which produce their respective outputs for vertices, and fragments outputted by the rasterization process. These are then compiled/linked together, where the vertex shader's output is used as one of the fragment shader's inputs.

Explain the Blinn-Phong shading model in your own words. Show a screenshot of your Blinn-Phong shader outputting only the ambient component, a screen shot only outputting the diffuse component, a screen shot only outputting the specular component, and one using the entire Blinn-Phong model.

Blinn-Phong shading gives the diffuse shading of an object, but also incorporates an ambient light component, as well as a specular reflection component. The ambient light is just ambient light in the environment, whereas the specular light deals with light when it is coming in a nearly mirror direction from the viewing perspective, as it would likely be underrepresented by the lambert cosine law used in diffuse lighting. It does this by using the half-angle vector based on the viewer direction and light direction, and the magnitude of the specular reflection depends on how close the normal and half-vectors are (using a dot-product). This value is then raised to a power depending on how reflective a surface is.
